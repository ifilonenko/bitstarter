How does it work?

Cryptunia uses public key cryptography to provide a secure channel to send messages. This method uses a public key to encrypt text, and a private key to decrypt text. So, when you want to receive send encrypted messages to someone, all you have to do is ask for his or her public key. The private key is never sent over the network, ensuring the security of the network. Unlike secret key (symmetric) encryption where both the sender and receiver have to know a password, public-key (asymmetric) encryption offers more robust security because the secret never has to be transmitted. Therefore, eavesdroppers will not be able to decrypt messages.

Cryptunia stores the private key in the browser storage (HTML 5 localStorage). Since the private key is extremely sensitive, it itself is encrypted with a password. This introduces two-factor authentication, so even if a potential attacker somehow gets the private key, he or she will still have to obtain the password to decrypt the private key.

With Cryptunia, your messages are so secure that even we cannot read them. Our code is completely open source and we follow a principle of radical transparency. Our code is available for everyone to see, so you can be assured that there are no backdoors and security researchers can document bugs so we can fix them.
